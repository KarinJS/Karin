# Karin v2.4 架构设计

> 把 v2.3 再拆干净。每个包只做一件事。command() 通过调用栈自动追踪来源。插件零 export。

---

## 一、v2.3 哪里还蠢

| 问题 | 说明 |
|------|------|
| **core.ts 是上帝模块** | 状态 + DSL + 调度 + bot 管理 + 生命周期 + 配置 + HMR 全塞一个文件。用函数代替 class 不代表解耦了，一个 200 行函数堆跟一个 200 行 class 一样糟 |
| **types 塞 core 内** | adapter 开发者只想要类型定义（Event/Element/Contact/AdapterApi），却必须依赖整个 karin 包含 dispatch/DSL/loader 全部运行时代码 |
| **command() 不知道谁调的** | `_file` 字段要手动传，HMR 的 removeByFile 依赖这个字段，但 DSL 没有自动填它。设计断裂 |
| **definePlugin + export default** | npm 插件需要 `export default definePlugin<T>((opts) => ({ name, start, stop }))`。插件开发者不该写这种框架仪式代码 |
| **Vite 模型配置** | `import onebot from 'karin-plugin-onebot'` + `plugins: [onebot({...})]`。10 个插件 = 10 行 import + 10 行调用。bot 用户不是前端工程师，这太重了 |
| **pipe 和 command 同一个包** | pipe 是框架级拦截器，command 是插件级 DSL，职责不同但混在一起 |
| **没有插件配置方案** | apps 插件如何读配置？npm 插件配置归谁管？v2.3 含糊地说"交给插件自己"，实际上什么都没提供 |

---

## 二、核心思想

### 2.1 调用栈溯源

当 `command()` 被调用时，从 `Error().stack` 读取调用者文件路径。**自动**。

```ts
// plugins/hello.ts
import { command } from 'karin'

command('hello', /^#hello$/, async (ctx) => {
  await ctx.reply('Hello!')
})
```

内部：

```ts
function command(name, pattern, fn, options?) {
  const file = getCallerFile()  // ← 从 Error().stack 解析
  commands.push({ name, pattern, fn, file, priority: options?.priority ?? 0 })
}
```

- 插件不需要 export 任何东西
- 插件不需要声明自己的名字
- HMR 时按 file 清除再重新 import，command() 重新注册新的 file
- npm 插件也一样 —— command() 知道调用来自 `node_modules/karin-plugin-xxx/dist/index.js`

### 2.2 包拆分原则

每个包**只做一件事**，自身代码量 < 100 行：

| 包 | 职责 | 代码量 |
|----|------|--------|
| `@karin/types` | 纯类型定义 | ~200（.d.ts） |
| `@karin/core` | dispatch + bot 管理 + pipe | ~80 |
| `@karin/dsl` | command / accept / handler / task / button + 调用栈溯源 | ~100 |
| `@karin/loader` | 插件扫描 + HMR | ~80 |
| `@karin/store` | 路径管理 | ✅ 已完成 |
| `@karin/sqlite` | KV 存储 | ✅ 已完成 |
| **`karin`** | 伞包：re-export 上面所有 + start/shutdown/defineConfig | ~40 |

用户只需 `import { command } from 'karin'`。内部怎么拆，用户不感知。

但 adapter 开发者可以只 `import { dispatch, registerBot } from '@karin/core'`，不拉 DSL 和 loader。

## 三、包详细设计

### 3.1 `@karin/types` — 纯类型

零运行时。adapter 和 plugin 共享的所有类型在这里。

```ts
// @karin/types/src/index.ts

// ─── 事件 ───
export interface Event {
  type: 'message' | 'notice' | 'request'
  subType?: string
  selfId: string
  userId: string
  contact: Contact
  timestamp: number
}

export interface MessageEvent extends Event {
  type: 'message'
  messageId: string
  rawMessage: string
  elements: Element[]
}

export interface NoticeEvent extends Event {
  type: 'notice'
  subType: string
}

export interface RequestEvent extends Event {
  type: 'request'
  subType: string
}

// ─── 消息元素 ───

  | { type: 'text'; text: string }
  | { type: 'image'; url: string }
  | { type: 'at'; userId: string }
  | { type: 'face'; id: number }
  | { type: 'reply'; messageId: string }
  | { type: 'forward'; messages: Element[][] }
  // 按需扩展

  peer: string
  subPeer?: string

  recallMessage(messageId: string): Promise<void>

  fn: (ctx: Context) => any
  priority: number
  file: string

  file: string

  file: string

  fn: (ctx: Context) => any
```

就是类型。没有任何运行时代码。

### 3.2 `@karin/core` — 调度引擎

依赖：`@karin/types`

职责：dispatch、bot 管理、pipe、共享状态数组。

```ts
// @karin/core/src/index.ts

import type {
  Event, MessageEvent, Context, AdapterApi,
  CommandEntry, AcceptEntry, HandlerEntry, ButtonEntry, Pipe
} from '@karin/types'

// ════ 共享状态（其他包可直接 import 读写） ════

export const bots      = new Map<string, AdapterApi>()
export const commands   : CommandEntry[]  = []
export const accepts    : AcceptEntry[]   = []
export const handlers   = new Map<string, HandlerEntry>()
export const buttons    = new Map<string, ButtonEntry>()
export const pipes      : Pipe[]  = []

// ════ Bot 管理 ════

export function registerBot(selfId: string, api: AdapterApi): void {
  bots.set(selfId, api)
}

export function unregisterBot(selfId: string): void {
  bots.delete(selfId)
}

export function getBot(selfId: string): AdapterApi | undefined {
  return bots.get(selfId)
}

export function getBots(): ReadonlyMap<string, AdapterApi> {
  return bots
}

// ════ Pipe 注册 ════

export function pipe(fn: Pipe): void {
  pipes.push(fn)
}

// ════ Dispatch ════

let cmdDirty = true
let accDirty = true

export function markDirty(type: 'command' | 'accept'): void {
  if (type === 'command') cmdDirty = true
  else accDirty = true
}

export async function dispatch(event: Event): Promise<void> {
  const bot = bots.get(event.selfId)
  if (!bot) return

  const ctx: Context = {
    event,
    bot,
    match: null,
    state: {},
    get userId()  { return event.userId },
    get selfId()  { return event.selfId },
    get contact() { return event.contact },
    reply(content) {
      const els = typeof content === 'string'
        ? [{ type: 'text' as const, text: content }]
        : content
      return bot.sendMessage(event.contact, els)
    },
  }

  // 管道
  for (const fn of pipes) {
    if (await fn(ctx) === false) return
  }

  // 匹配
  if (event.type === 'message') {
    if (cmdDirty) { commands.sort((a, b) => b.priority - a.priority); cmdDirty = false }
    const msg = (event as MessageEvent).rawMessage
    for (const cmd of commands) {
      const m = cmd.pattern.exec(msg)
      if (!m) continue
      ctx.match = m
      try { await cmd.fn(ctx) } catch (e) { console.error(`[cmd:${cmd.name}]`, e) }
      return
    }
  } else {
    if (accDirty) { accepts.sort((a, b) => b.priority - a.priority); accDirty = false }
    for (const acc of accepts) {
      if (acc.event !== '*' && acc.event !== event.subType) continue
      try { await acc.fn(ctx) } catch (e) { console.error('[accept]', e) }
    }
  }
}

// ════ 按文件清除（HMR 用） ════

export function removeByFile(file: string): void {
  spliceBy(commands, file)
  spliceBy(accepts, file)
  for (const [k, v] of handlers) { if (v.file === file) handlers.delete(k) }
  for (const [k, v] of buttons)  { if (v.file === file) buttons.delete(k)  }
  cmdDirty = true
  accDirty = true
}

function spliceBy(arr: { file: string }[], file: string): void {
  for (let i = arr.length - 1; i >= 0; i--) {
    if (arr[i].file === file) arr.splice(i, 1)
  }
}
```

**~80 行。** 纯逻辑，无 IO。

### 3.3 `@karin/dsl` — 五个注册函数 + 调用栈溯源

依赖：`@karin/core`（往共享数组里 push）、`@karin/types`

```ts
// @karin/dsl/src/index.ts

import {
  commands, accepts, handlers, buttons, markDirty
} from '@karin/core'

import type {
  Context, CommandEntry, AcceptEntry
} from '@karin/types'

// ════ 调用栈溯源 ════

function getCallerFile(): string {
  const err = { stack: '' }
  Error.captureStackTrace(err, getCallerFile)
  // 跳过当前函数和 DSL 函数，取真正的调用者
  const lines = err.stack.split('\n')
  for (const line of lines) {
    const match = line.match(/\(?file:\/\/\/(.+?)[\?:)]/)
      ?? line.match(/\(?(.+?\.[tj]s)[\?:)]/)
    if (match && !match[1].includes('@karin/dsl') && !match[1].includes('@karin\\dsl')) {
      return match[1].replaceAll('\\', '/')
    }
  }
  return 'unknown'
}

// ════ DSL ════

export function command(
  name: string,
  pattern: RegExp | string,
  fn: (ctx: Context) => any,
  options?: { priority?: number }
): void {
  const file = getCallerFile()
  const regex = typeof pattern === 'string' ? new RegExp(pattern) : pattern
  commands.push({ name, pattern: regex, fn, priority: options?.priority ?? 0, file })
  markDirty('command')
}

export function accept(
  event: string | '*',
  fn: (ctx: Context) => any,
  options?: { priority?: number }
): void {
  const file = getCallerFile()
  accepts.push({ event, fn, priority: options?.priority ?? 0, file })
  markDirty('accept')
}

export function handler(key: string, fn: (ctx: Context) => any): void {
  const file = getCallerFile()
  handlers.set(key, { key, fn, file })
}

export function task(name: string, cron: string, fn: () => any): void {
  const file = getCallerFile()
  // 内部启动 cron 调度（或由外部 scheduler 插件消费 tasks 数组）
  ;(globalThis as any).__karin_tasks ??= []
  ;(globalThis as any).__karin_tasks.push({ name, cron, fn, file })
}

export function button(key: string, fn: (ctx: Context) => any): void {
  const file = getCallerFile()
  buttons.set(key, { key, fn, file })
}
```

**~60 行。** 每个 DSL 函数就是往数组里 push，顺带自动读 file。

### 3.4 `@karin/loader` — 扫描 + HMR

依赖：`@karin/core`（removeByFile）

```ts
// @karin/loader/src/index.ts

import fs from 'node:fs'
import path from 'node:path'
import { pathToFileURL } from 'node:url'
import { removeByFile } from '@karin/core'

// ════ 加载 apps 插件 ════

export async function loadApps(dir: string): Promise<void> {
  if (!fs.existsSync(dir)) return
  const files = collectFiles(dir)
  for (const file of files) {
    await importFile(file)
  }
}

function collectFiles(dir: string): string[] {
  const result: string[] = []
  for (const entry of fs.readdirSync(dir, { withFileTypes: true })) {
    if (entry.isFile() && /\.[tj]s$/.test(entry.name)) {
      result.push(path.join(dir, entry.name))
    }
  }
  return result
}

async function importFile(file: string): Promise<void> {
  try {
    await import(`${pathToFileURL(file)}?t=${Date.now()}`)
  } catch (err) {
    console.error(`[loader] ${path.basename(file)}:`, err)
  }
}

// ════ 加载 npm 插件 ════

export async function loadNpmPlugins(
  entries: (string | [string, Record<string, any>])[]
): Promise<{ start?(): any; stop?(): any }[]> {
  const instances: { start?(): any; stop?(): any }[] = []
  for (const entry of entries) {
    const [name, options] = Array.isArray(entry) ? entry : [entry, undefined]
    try {
      // 设置当前加载上下文（让 pluginConfig() 能读到配置）
      currentLoading = { name, options: options ?? {} }
      const mod = await import(name)
      // 如果插件 default export 是函数且有配置，调用它
      if (typeof mod.default === 'function' && options !== undefined) {
        const inst = mod.default(options)
        if (inst && typeof inst === 'object') instances.push(inst)
      }
    } catch (err) {
      console.error(`[loader] ${name}:`, err)
    } finally {
      currentLoading = null
    }
  }
  return instances
}

// ════ 插件配置上下文 ════

let currentLoading: { name: string; options: Record<string, any> } | null = null

/**
 * npm 插件在顶层调用此函数读取传入的配置。
 * 内部通过加载上下文获取，无需 export 任何东西。
 */
export function pluginConfig<T = Record<string, any>>(): T {
  return (currentLoading?.options ?? {}) as T
}

/**
 * 读取当前正在加载的插件名称
 */
export function pluginName(): string {
  return currentLoading?.name ?? 'unknown'
}

// ════ HMR（仅 apps） ════

export function watchApps(dir: string): void {
  if (!fs.existsSync(dir)) return
  fs.watch(dir, { recursive: false }, async (_event, filename) => {
    if (!filename || !/\.[tj]s$/.test(filename)) return
    const file = path.join(dir, filename)
    if (!fs.existsSync(file)) return

    removeByFile(file.replaceAll('\\', '/'))
    await importFile(file)
    console.log(`[hmr] ${filename}`)
  })
}
```

**~80 行。**

### 3.5 `karin` — 伞包

依赖：所有上面的包

```ts
// karin/src/index.ts

// re-export 全部 API（用户只需 import { xxx } from 'karin'）
export { dispatch, registerBot, unregisterBot, getBot, getBots, pipe } from '@karin/core'
export { command, accept, handler, task, button } from '@karin/dsl'
export { pluginConfig, pluginName } from '@karin/loader'
export type * from '@karin/types'

// ════ 框架入口 ════

import { loadApps, loadNpmPlugins, watchApps } from '@karin/loader'

import type { KarinConfig } from '@karin/types'

let config: KarinConfig = {}
let npmInstances: { stop?(): any }[] = []

// 加载配置
const CONFIG_NAMES = ['karin.config.ts', 'karin.config.js', 'karin.config.mjs']

async function loadConfig(): Promise<KarinConfig> {
  const { resolve } = await import('node:path')
  for (const name of CONFIG_NAMES) {
    try {
      const mod = await import(resolve(process.cwd(), name))
      return mod.default ?? mod
    } catch { continue }
  }
  return {}
}

export async function start(): Promise<void> {
  config = await loadConfig()

  // 加载 npm 插件
  if (config.plugins?.length) {
    npmInstances = await loadNpmPlugins(config.plugins)
    for (const inst of npmInstances) {
      await inst.start?.()
    }
  }

  // 加载 apps 插件
  const pluginsDir = (await import('node:path')).resolve(process.cwd(), 'plugins')
  await loadApps(pluginsDir)
  watchApps(pluginsDir)

  const { commands, bots } = await import('@karin/core')
  console.log(`[karin] ready · ${commands.length} commands · ${bots.size} bots`)
}

export async function shutdown(): Promise<void> {
  for (const inst of npmInstances) {
    await inst.stop?.()
  }
}

export function useConfig(): Readonly<KarinConfig> {
  return config
}

export function defineConfig(cfg: KarinConfig): KarinConfig {
  return cfg
}
```

## 四、npm 插件 — 零 export 写法

### 4.1 简单插件（纯 DSL，无需配置）

```ts
// karin-plugin-hello/src/index.ts
import { command } from 'karin'

command('hello', /^#hello$/, async (ctx) => {
  await ctx.reply('Hello from npm plugin!')
})
```

**一行 import，一个 command()。没有 export，没有 definePlugin，没有 start/stop。**

karin.config.ts 中：

```ts
plugins: ['karin-plugin-hello']
```

框架 `await import('karin-plugin-hello')` → command() 自动注册 → 完事。

### 4.2 带配置的插件（用 pluginConfig）

```ts
// karin-plugin-weather/src/index.ts
import { command, pluginConfig } from 'karin'

interface WeatherConfig {
  apiKey: string
  defaultCity?: string
}

const config = pluginConfig<WeatherConfig>()

command('weather', /^#天气\s*(.*)$/, async (ctx) => {
  const city = ctx.match![1] || config.defaultCity || '北京'
  const data = await fetch(`https://api.example.com/weather?city=${encodeURIComponent(city)}&key=${config.apiKey}`)
  const json = await data.json()
  await ctx.reply(`${city}: ${json.temp}°C`)
})
```

karin.config.ts 中：

```ts
plugins: [
  ['karin-plugin-weather', { apiKey: 'xxx', defaultCity: '上海' }],
]
```

**还是没有 export。** `pluginConfig()` 通过加载上下文（loader 在 import 前设置的）拿到配置。

### 4.3 适配器插件（需要生命周期）

适配器需要主动建连、监听断连，所以需要 start/stop。这是唯一需要 default export 的场景——不是为了"声明自己是插件"，是为了返回生命周期钩子。

```ts
// karin-plugin-onebot/src/index.ts
import { registerBot, unregisterBot, dispatch, pluginConfig } from 'karin'
import type { AdapterApi, Contact, Element, Event } from 'karin'
import { WebSocketServer, WebSocket } from 'ws'

interface OneBotConfig {
  wsServer?: { enable?: boolean; port?: number }
  wsClient?: { url: string; token?: string }[]
}

const config = pluginConfig<OneBotConfig>()
const connections = new Map<string, WebSocket>()

// default export 返回生命周期（可选）
export default () => ({
  start() {
    if (config.wsServer?.enable) {
      const wss = new WebSocketServer({ port: config.wsServer.port ?? 6700 })
      wss.on('connection', (ws, req) => {
        const selfId = req.headers['x-self-id'] as string
        const api = makeApi(ws)
        registerBot(selfId, api)
        connections.set(selfId, ws)

        ws.on('message', raw => {
          const event = transform(JSON.parse(raw.toString()), selfId)
          if (event) dispatch(event)
        })
        ws.on('close', () => {
          unregisterBot(selfId)
          connections.delete(selfId)
        })
      })
    }
  },

  stop() {
    for (const [id, ws] of connections) {
      ws.close()
      unregisterBot(id)
    }
  },
})

function makeApi(ws: WebSocket): AdapterApi {
  return {
    async sendMessage(contact: Contact, elements: Element[]) {
      // ... 转换为 OneBot 格式并发送
      return 'msg_id'
    },
    async recallMessage(messageId: string) {
      ws.send(JSON.stringify({ action: 'delete_msg', params: { message_id: messageId } }))
    },
  }
}
```

注意：即使有 `export default`，里面的 `pluginConfig()` 仍然是零仪式获取配置。export 只用于返回 start/stop，不是为了注册。

### 4.4 pipe 插件

## 五、apps 插件 — 最简形态

```ts
// plugins/hello.ts
import { command } from 'karin'

command('hello', /^#hello$/,  ctx => ctx.reply('Hello!'))
```

三行，三个命令。这是 karin 提供的最短路径。

HMR：修改文件 → loader 调 `removeByFile('plugins/hello.ts')` 清除旧注册 → 重新 import → command() 重新注册新的（file 自动填充）。

## 六、karin.config.ts

```ts
// karin.config.ts
import { defineConfig } from 'karin'

export default defineConfig({
  server: {
    port: 7777,
    host: '0.0.0.0',
  },

  logger: {
    level: 'info',
    daysToKeep: 7,
  },

  permissions: {
    master: ['123456'],
    admin: ['789012'],
  },

  plugins: [
    // 无配置：字符串
    'karin-plugin-permissions',

    // 带配置：元组 [name, options]
    ['karin-plugin-onebot', {
      wsServer: { enable: true, port: 6700 },
      wsClient: [{ url: 'ws://127.0.0.1:8080' }],
    }],

    ['karin-plugin-webui', { port: 7777 }],

    ['karin-plugin-blacklist', {
      users: ['spam-user-1'],
      groups: ['bad-group-1'],
    }],
  ],
})
```

- 框架配置 + 插件列表在同一个文件
- 插件配置是元组第二项，框架不关心结构
- 数组顺序 = 加载顺序（pipe 插件放前面）
- 类型安全（defineConfig 提供顶层类型，插件配置各插件自己定义）

## 七、包依赖图

```
@karin/types        ← 零依赖，纯 .d.ts
    ↑
@karin/core         ← 仅依赖 types
    ↑
@karin/dsl          ← 依赖 core（导入共享数组）+ types
    ↑
@karin/loader       ← 依赖 core（removeByFile）
    ↑
  karin             ← 伞包，re-export 全部

@karin/store        ← 独立，零依赖
@karin/sqlite       ← 依赖 store + better-sqlite3
```

```
adapter 插件  → 仅需 karin（实际只用 core 部分：dispatch, registerBot）
command 插件  → 仅需 karin（实际只用 dsl 部分：command, accept）
pipe 插件     → 仅需 karin（实际只用 core 部分：pipe）
```

**adapter 不依赖 dsl。dsl 不依赖 loader。每一层只看到它需要的。**

## 八、调用全链路

### 消息处理

```
1. WebSocket 收到原始数据
   └─ karin-plugin-onebot 内部处理

2. 转换为标准 Event
   └─ const event: MessageEvent = transform(raw)

3. 调用 dispatch
   └─ dispatch(event)
      │
      ├─ 构建 ctx
      │
      ├─ for (pipe of pipes)        ← 权限检查、黑名单过滤
      │    if pipe(ctx) === false → return
      │
      ├─ sort commands (if dirty)
      │
      └─ for (cmd of commands)      ← 正则匹配
           cmd.pattern.exec(msg) → hit!
           └─ await cmd.fn(ctx)
              └─ ctx.reply('Hello!')
                 └─ bot.sendMessage(contact, elements)
                    └─ ws.send(...)
```

### 插件加载

```
1. karin.start()

2. loadConfig()
   └─ import('karin.config.ts')

3. loadNpmPlugins(config.plugins)
   └─ for ['karin-plugin-onebot', { wsServer: { enable: true } }]
      │
      ├─ 设置 currentLoading = { name, options }
      ├─ await import('karin-plugin-onebot')
      │    └─ 顶层代码执行：
      │       ├─ pluginConfig() → 读取 currentLoading.options
      │       ├─ command() → 自动 getCallerFile() → push 到 commands
      │       └─ export default () => ({ start, stop })
      ├─ 调用 default(options) 获取 { start, stop }
      └─ currentLoading = null

4. 调用所有 npm 插件的 start()
   └─ onebot.start() → 启动 WebSocket Server

5. loadApps('plugins/')
   └─ for plugins/hello.ts
      └─ await import('plugins/hello.ts?t=xxx')
         └─ 顶层 command() → getCallerFile() = 'plugins/hello.ts'

6. watchApps('plugins/')
   └─ fs.watch → 文件变化 → removeByFile → re-import
```

## 十一、设计原则

1. **调用栈即来源** — command() 自动知道谁注册的，零仪式
2. **一包一事** — types 管类型，core 管调度，dsl 管注册，loader 管加载
3. **零 export 插件** — 插件文件写 command() 就完事，不需要 export 任何东西
4. **配置即数据** — karin.config.ts 里字符串或元组，不需要 import 插件包
5. **依赖单向下沉** — types ← core ← dsl / loader ← karin，没有环
6. **每个包 < 100 行** — 超了说明职责不纯，要拆
