# ESM 动态导入错误演示

这个文件夹包含了演示 ESM 动态导入各种错误情况的示例代码。

## 文件说明

### `dynamic-import-demo.mjs`

主演示文件，包含以下错误测试：

1. **模块不存在错误** - 尝试导入不存在的模块
2. **语法错误** - 导入包含语法错误的模块
3. **缺失依赖错误** - 导入依赖不存在包的模块
4. **循环依赖错误** - 导入存在循环依赖的模块
5. **权限错误** - 尝试访问受限制的文件
6. **网络错误** - 尝试从无效的网络地址导入

### `syntax-error-module.mjs`

包含语法错误的模块，用于演示语法解析失败的情况。

### `missing-dependency-module.mjs`

尝试导入不存在的 npm 包的模块，用于演示依赖缺失的情况。

### `circular-a.mjs` & `circular-b.mjs`

相互导入的模块，用于演示循环依赖的情况。

## 运行方式

```bash
node dynamic-import-demo.mjs
```

## 预期输出

每个测试都会捕获并显示相应的错误信息，包括：

- 错误类型（constructor name）
- 错误信息（message）
- 错误代码（code，如果有）
- 错误堆栈（stack，部分情况）

这些示例可以帮助理解在插件系统中处理动态导入错误时需要考虑的各种情况。
